# MemoryTools介绍
MemoryTools 是一个用于在 OpenEuler 系统中分析内存性能的工具集。它包括了一系列的工具，可以用于监控、分析内存的使用情况。
具体能够获得内存的这几点信息：
1. 物理内存状态：包括物理内存总量、已用量和占用率。
2. 可用内存：当前系统可用的物理内存量。
3. 交换分区状态：包括交换分区总量、已用量和占用率。
4. 内存警报检查：根据设定的阈值，判断当前内存使用率是否超过警报线。

## 涉及到的psutil工具及其返回的内容
1. psutil.virtual_memory() 用于获取物理内存的整体统计信息。
```输出示例
svmem(total=17179869184, available=8589934592, percent=50.0, used=8589934592, free=8589934592, active=0, inactive=0, buffers=0, cached=0, shared=0, slab=0)
总内存: 16.00 GB
可用内存: 8.00 GB
内存使用率: 50.0%
已用内存: 8.00 GB
空闲内存: 8.00 GB
活跃内存: N/A
不活跃内存: N/A
缓冲区: N/A
缓存: N/A
```
2. psutil.swap_memory() 用于获取交换分区的整体统计信息。
```输出示例
sswap(total=2147483648, used=536870912, free=1610612736, percent=25.0, sin=0, sout=0)
交换分区总量: 2.00 GB
交换分区已用: 0.50 GB
交换分区空闲: 1.50 GB
交换分区使用率: 25.0%
```
3. psutil.virtual_memory().shared 用于获取共享内存信息（主要在 Linux 上）。
```输出示例
共享内存: 0.00 B
```
4. psutil.Process.memory_info() 用于获取特定进程的内存使用详情。
```输出示例
pmem(rss=10485760, vms=20971520, pfaults=1000, pageins=0)
RSS (Resident Set Size): 10.00 MB
VMS (Virtual Memory Size): 20.00 MB
```
